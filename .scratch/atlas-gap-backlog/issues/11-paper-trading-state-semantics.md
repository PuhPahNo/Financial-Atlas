# Paper Trading State Semantics

Status: implemented

## Problem

Seeded models, illustrative metrics, newly saved configs, and real backtest results can read as the
same level of evidence if labels are not explicit.

## Scope

- Audit labels for seeded, cloned, saved, backtested, and archived states.
- Add empty/loading/error states for all major Paper Trading panels.
- Make destructive or irreversible actions explicit.

## Acceptance Criteria

- A user can tell whether a metric is seeded illustrative data or generated by their backtest.
- Empty profiles and empty strategy lists have useful next actions.
- Deletion/archive behavior matches historical-run preservation.

## Implementation Notes

- Added `metricState` to adapted Paper Trading models: `backtested`, `seeded_illustrative`, or
  `custom_projection`.
- Model cards and detail views now show persistent metric-source badges with explanatory tooltips.
- Archive confirmation copy now matches backend behavior: models/traders leave active lists while
  historical backtests, account allocations, and stored snapshots remain preserved.
- Archive action titles/toasts now use archive language instead of destructive delete language.
- Existing empty states remain in place for no models/no traders/no fills/no allocations.

## QA Evidence

- `npm run build` in `frontend/` -> passed.
- Browser smoke on `http://localhost:3000/paper-trading` verified model cards show persistent
  `Backtested` metric-source badges. The local DB has warmed headline backtests for seeded models,
  so seeded/projection badges are represented by code paths rather than visible in this data state.

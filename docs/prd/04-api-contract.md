# 04 — API Contract

> Parent: [00-master-prd.md](00-master-prd.md) · The stable boundary between frontend and backend.
> Endpoints here are referenced (not redefined) by each feature PRD.

## 1. Purpose / why

Define a versioned, predictable REST surface with a single error model and explicit
preconditions/postconditions, so the frontend can be built against a contract and the backend can
change providers/storage freely behind it.

## 2. User stories & acceptance criteria

- *As the frontend dev,* every endpoint has a documented request/response schema. **AC:** OpenAPI is
  auto-generated by FastAPI and matches the shapes here.
- *As a client,* errors are uniform and machine-readable. **AC:** all errors use the envelope in §6.
- *As a client,* I can tell when data is stale or which provider served it. **AC:** responses carry
  `meta.served_by` and `meta.stale`.

## 3. Scope (in / out)

- **In:** route list, request params, response envelope, error model, versioning, pagination.
- **Out:** business logic (feature PRDs), persistence ([03](03-data-model.md)).

## 4. Conventions

- Base path: `/api/v1`. JSON only. `snake_case` fields. ISO-8601 dates. Money as numbers + a sibling
  `currency` field at the document level.
- Every successful response is wrapped:

```json
{ "data": { /* payload */ },
  "meta": { "ticker": "AAPL", "served_by": "sec_edgar", "stale": false,
            "as_of": "2026-06-01T00:00:00Z" } }
```

- **DBC per endpoint:** documented **preconditions** (valid ticker, param ranges) and
  **postconditions** (shape, ordering, `meta` populated). Validation via Pydantic request/response
  models; violations → typed errors.

## 5. Endpoints

### Company & search
```txt
GET /api/v1/search?q={query}                 → [{ticker,name,exchange}]  (typeahead)
GET /api/v1/company/{ticker}                 → CompanyProfile + key metrics
```

### Prices
```txt
GET /api/v1/prices/{ticker}?range=1y&interval=1d
    range ∈ {1m,3m,6m,1y,3y,5y,max}; interval ∈ {1d,1wk,1mo}
                                             → { bars:[OHLCV], currency }
```

### Financial statements
```txt
GET /api/v1/financials/{ticker}/income?period=annual|quarter&limit=N
GET /api/v1/financials/{ticker}/balance-sheet?period=...
GET /api/v1/financials/{ticker}/cash-flow?period=...
                                             → { statements:[...], currency }
GET /api/v1/financials/{ticker}/cash-flow-analysis?period=...
                                             → derived FCF metrics (see 13)
```

### Valuation
```txt
GET  /api/v1/valuation/{ticker}              → default multi-model valuation (see 14)
POST /api/v1/valuation/{ticker}              → custom assumptions → recomputed valuation
     body: { weights?, scenarios?, assumptions:{ growth_rate_1_5, growth_rate_6_10,
             discount_rate, terminal_growth_rate, fair_pe, fair_ev_ebitda, fair_ev_sales, ... } }
GET  /api/v1/valuation/{ticker}/history      → stored valuation_results over time
```

### Ownership & insiders
```txt
GET /api/v1/ownership/{ticker}/insiders?since={date}     → insider transactions + summary (see 16)
GET /api/v1/ownership/{ticker}/institutions?as_of={date} → 13F holders + QoQ changes (see 17)
```

### Filings
```txt
GET /api/v1/filings/{ticker}?forms=10-K,10-Q,8-K&limit=N → filings index (see 18)
GET /api/v1/filings/search?q={text}&ticker={ticker}      → EDGAR full-text search proxy
```

### Peers, screener, watchlists, news/macro
```txt
GET  /api/v1/peers/{ticker}                  → peer set + median multiples (see 15)
POST /api/v1/screener                        → filter spec → matching tickers + metrics (see 20)
GET  /api/v1/watchlists                      → lists; POST/PUT/DELETE for CRUD (see 21)
GET  /api/v1/news/{ticker}                   → news + earnings/dividend calendar (see 19)
GET  /api/v1/macro?series=DGS10,CPIAUCSL     → FRED series (see 19)
```

### Paper trading and backtesting
```txt
GET  /api/v1/paper-trading/categories        → categories + seeded/user strategies
GET  /api/v1/paper-trading/strategies        → strategy list
POST /api/v1/paper-trading/strategies        → create user strategy
GET  /api/v1/paper-trading/strategies/{id}   → strategy detail
PUT  /api/v1/paper-trading/strategies/{id}   → update user strategy
POST /api/v1/paper-trading/strategies/{id}/clone → clone seeded/user strategy
DELETE /api/v1/paper-trading/strategies/{id} → archive/delete user strategy
POST /api/v1/backtests                       → run strategy backtest (see 23)
GET  /api/v1/backtests/{id}                  → persisted run, trades, equity curve
POST /api/v1/paper-trading/portfolios        → create simulated portfolio
GET  /api/v1/paper-trading/portfolios        → list simulated portfolios
GET  /api/v1/paper-trading/portfolios/{id}   → positions, orders, fills, snapshots
```

### Research assistant
```txt
POST /api/v1/assistant/sessions              → create chat session
GET  /api/v1/assistant/sessions/{id}         → messages + pending actions
POST /api/v1/assistant/sessions/{id}/messages → chat turn, may return pending action
POST /api/v1/assistant/actions/{id}/confirm  → execute pending state-changing tool
POST /api/v1/assistant/actions/{id}/reject   → reject pending action
```

## 6. Error model

```json
{ "error": { "code": "RATE_LIMITED", "message": "All price providers exhausted",
             "ticker": "AAPL", "retry_after_seconds": 30 } }
```

| HTTP | `code` | Cause |
| --- | --- | --- |
| 400 | `INVALID_REQUEST` | bad params (precondition failed) |
| 404 | `NOT_FOUND` | ticker/CIK unresolved or no data |
| 429 | `RATE_LIMITED` | all providers for the domain rate-limited |
| 503 | `PROVIDER_UNAVAILABLE` | upstream down, no usable cache |
| 500 | `INTERNAL` | unexpected (logged with request id) |

Maps from the typed exceptions in [01 §7](01-architecture.md).

## 7. Versioning & pagination

- Version in path (`/v1`). Additive changes only within a version; breaking → `/v2`.
- List endpoints accept `limit`/`cursor`; default and max `limit` documented per endpoint.

## 8. Dependencies

All feature PRDs (10–24) consume these routes; [01](01-architecture.md) (routers),
[05](05-caching-and-jobs.md) (`meta.stale`/`served_by` sourced from cache+fallback).

## 9. Edge cases & error handling

- Partial data (e.g. prices but no fundamentals) → `200` with present sections populated and a
  `meta.warnings` array; never fabricate.
- Unknown query param → ignored (forward-compatible), documented in OpenAPI.

## 10. Testing requirements

- Schema tests: responses validate against the Pydantic response models.
- Error-mapping tests: each typed exception yields the right HTTP + `code`.
- Contract snapshot of OpenAPI checked in; diff reviewed on change.

## 11. Open questions & assumptions

- Auth header reserved (`Authorization`) but unused while single-tenant; added in [30](30-deployment-render.md).

## 12. Done criteria

- Tracer bullet: `/search`, `/company/{ticker}`, `/prices/{ticker}` implemented with the envelope +
  error model and OpenAPI generated. → Thicken endpoint by endpoint with each feature phase.
